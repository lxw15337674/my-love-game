# Robot War Prototype 数值 Playtest 记录口径

> 目标：先统一记录口径，不直接改难度曲线。每次调数值前后都按同一表记录，避免只靠感觉乱砍。

## 记录波次

固定记录：wave 1 / 3 / 6 / 9 / 12。

- wave 1：首关入门压力。
- wave 3：第一大关 Boss。
- wave 6：第二大关 Boss，检验二关升压。
- wave 9：第三大关 Boss，检验中期成型要求。
- wave 12：第四大关 Boss，检验后续压力是否过硬。

## 每波记录字段

| 字段 | 记录内容 |
| --- | --- |
| 起始构筑 | 武器、护盾、模块数量、模块槽等级、关键联动 |
| 起始资源 | 生命、护盾、材料、危险等级 |
| 结束结果 | 通过 / 死亡 / 超时 / Boss 未击破 |
| 击杀数 | 该波结算击杀数 |
| 剩余生存 | 结束时生命、护盾 |
| 材料收入 | 本波获得材料、结算后材料总数 |
| 商店购买能力 | 结算后能买几件核心商品、是否还能刷新/升级模块槽 |
| 压力备注 | 死因、最危险敌人、是否被 HUD/特效遮挡 |

## 当前配置基线（非实战结论）

| Wave | 结构节点 | 预期关注点 |
| --- | --- | --- |
| 1 | 普通小关 | 入门敌量、基础武器能否稳定清杂 |
| 3 | 第一大关 Boss | Boss 血量/护盾是否让新手卡死 |
| 6 | 第二大关 Boss | 二关升压后是否需要已有模块/第二武器 |
| 9 | 第三大关 Boss | 中期敌人压力和经济是否支撑构筑 |
| 12 | 第四大关 Boss | 后期压力是否过早硬化 |

## 判定口径

- 若 wave 1 经常低血或死亡：入门敌量/初始武器/首关奖励过硬。
- 若 wave 3 Boss 击破困难但普通关轻松：Boss 血量或护盾曲线过高。
- 若 wave 6 起必须靠特定武器才稳：构筑入口或商店武器权重可能过窄。
- 若 wave 9/12 材料不足以买核心商品：经济或商店价格曲线需复查。
- 若死亡主要来自看不清危险：先修 UX/特效可见性，不先调数值。

## 下一步

完成低风险 UX 小修后，按此表做一次人工/半自动 playtest 记录，再决定是否调整 `average_run_target_wave`、敌人压力、经济奖励或补给转轮成本。
